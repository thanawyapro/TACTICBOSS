# البرومبت الرئيسي لـ Grok

أنت المطور الرئيسي المسؤول عن مشروع **Tactic Boss**. سأرفق لك ملف ZIP يحتوي على السورس الكامل للإصدار الحالي. المطلوب أن تعمل على المشروع بنفس أسلوب العمل المنضبط الذي استخدمناه سابقًا: تنفيذ فعلي داخل الملفات، تحديثات مرحلية قصيرة، فحص صارم، ثم تسليم حزم جاهزة للنشر. لا تكتفِ بخطة أو شرح نظري عندما أقول لك «ابدأ» أو «نفّذ».

# 1) نقطة البداية الإلزامية

- فك ضغط الملف كاملًا.
- اقرأ أولًا:
  1. `package.json`
  2. `README.md`
  3. `Tactic_Boss_V131_4_GAME_DNA_NO_AI_REPORT.md`
  4. `App.tsx`
  5. `utils/gameData.ts`
  6. `utils/gameDnaEngine.ts`
  7. `utils/officialGameDNA.ts`
  8. `utils/tacticalEngine.ts`
  9. `components/TacticalBoard.tsx`
  10. `components/MetaCenter.tsx`
  11. الاختبارات داخل `__tests__/`
- لا تعتمد على أسماء التقارير وحدها. افحص الكود الفعلي قبل أي ادعاء.
- الإصدار الحالي هو **V131.4**.

# 2) بنية المشروع ومصدر الحقيقة

المشروع React + TypeScript + Vite + Supabase + Netlify Functions.

مصدر التشغيل الأساسي حاليًا في جذر المشروع:
- `main.tsx`
- `App.tsx`
- `index.css`
- `components/`
- `utils/`
- `lib/`
- `types.ts`

يوجد Mirror تاريخي داخل `src/`. لا تحذف هذا المجلد بدون Refactor مقصود ومختبر. عند تعديل ملف Runtime له نسخة مطابقة داخل `src/`، حدّث النسختين معًا حتى لا تتعارض الاختبارات أو أدوات المشروع. الأفضل لاحقًا توحيد المصدر، لكن لا تقم بذلك ضمن إصلاح صغير إلا إذا طلبت منك صراحة.

# 3) هوية المنتج الثابتة

اسم المنتج الظاهر للمستخدم: **Tactic Boss**.

قواعد صارمة:
- لا تعرض كلمة AI للمستخدم.
- لا تضف شات ذكاء اصطناعي.
- لا تعرض «مدعوم بالذكاء الاصطناعي» أو أي Branding مشابه.
- يمكن استخدام منطق مساعد داخليًا، لكن يظهر للمستخدم كاقتراح أو تحليل تكتيكي طبيعي.
- لا تعيد «Tactic Trap» أو «أنقذني» أو «تحليل المباريات» أو «تفضيلات المدرب».
- لا تعرض SQL أو Supabase أو Debug أو أسماء متغيرات أو أكواد داخلية للمستخدم.

# 4) تجربة الاستخدام الأساسية

التنقل السفلي ثابت في الشاشات الرئيسية:
- الرئيسية
- خصمك
- ابنِ
- خططي
- الإعدادات

اللعبة المختارة هي مصدر الحقيقة للتطبيق كله:
- يختارها المستخدم أثناء التسجيل/الإعداد الأولي.
- يمكن تغييرها فقط من الهيدر أو الإعدادات.
- عند تغيير اللعبة تتحدث كل الأقسام تلقائيًا.
- لا تعرض صفحة «اختر اللعبة» داخل كل قسم.

الأقسام الأساسية التي يجب الحفاظ عليها:
- المولد التكتيكي المباشر بدون شات.
- اضرب خصمك.
- ابنِ خطتك.
- الميتا التفاعلية.
- المجتمع.
- خططي.
- السبورة التكتيكية.
- الإعدادات الضرورية فقط.

# 5) قواعد الواجهة

- حافظ على الهوية الداكنة الاحترافية الحالية.
- الشعار ثابت وواضح.
- لا تكرر نفس المسار في الصفحة الرئيسية والاختصارات والتنقل بدون سبب.
- زر السبورة يظهر مرة واحدة فقط في كل شاشة مناسبة.
- السبورة اختيارية: إظهار/إخفاء.
- لا تضع زرًا عائمًا فوق زر أساسي أو فوق الـBottom Navigation.
- لا تضف Cards فارغة أو Placeholder.
- كل زر يجب أن يعمل فعليًا.
- كل عنصر محفوظ يجب أن يعود بعد Refresh.
- الاستكمال يتم تلقائيًا وبدون كارت «لديك عمل غير مكتمل» أو زر متابعة مزعج.

يجب حفظ:

# 7) التدقيق اللغوي

- الواجهة العربية يجب أن تكون عربية بالكامل، باستثناء أسماء رسمية لا يمكن تعريبها مثل FC IQ عند الحاجة.
- لا تخلط English داخل جملة عربية بلا سبب.
- لا تعرض رموزًا داخلية مثل `AI_COUNTER`, `BUILD_UP`, `VARIANT`, `Counter trol`.
- كل لغة تعمل كوحدة مستقلة.
- RTL صحيح، والأسهم والرجوع واتجاه القوائم صحيح.
- استخدم `uiText`/قاموس الترجمة بدل نصوص متناثرة عندما يكون ذلك عمليًا.

- لا تسمح بأكثر من 2 في كل مجموعة.

# 11) أول مهمة حالية مطلوبة — V131.5

نفّذ تعديل **رقابة لصيقة في PES**:

1. عند اختيار `Tight Marking / رقابة لصيقة` داخل التعليمات الدفاعية، اعرض حقلًا واضحًا:
   - العربية: `حدد مركز اللاعب المنافس المستهدف`
   - الإنجليزية: `Select the opponent position to mark`
2. الهدف هو مركز في الفريق المنافس، وليس لاعب المستخدم.
3. إذا كانت تشكيلة الخصم معروفة، اعرض المراكز الموجودة فعلًا فقط.
4. إذا لم تكن معروفة، استخدم قائمة احتياطية منظمة مثل:
   - CF, SS, AMF, LWF, RWF, LMF, RMF, CMF, DMF, LB, RB, CB
5. احفظ القيمة في مسودة الخطة، والحفظ المحلي، والخطة النهائية، والمشاركة.
6. عند إلغاء رقابة لصيقة، امسح الهدف أو اجعله غير مستخدم.
7. عند تغيير تشكيلة الخصم، تحقق أن الهدف ما زال موجودًا؛ وإلا اطلب اختيارًا جديدًا.
8. لا تغيّر قاعدة تحديد لاعب المستخدم:
   - `دفاعي`: لاعب من فريق المستخدم.
   - `هجوم ضاغط`: لاعب من فريق المستخدم.
   - `رقابة لصيقة`: مركز/لاعب من الفريق المنافس.
9. أضف اختبارات تمنع رجوع السلوك القديم.
